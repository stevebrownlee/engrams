# MCP Messaging Guidelines for Engrams

## Quick Reference: The 6 Core Messages

Use these messages consistently across all marketing, documentation, and communication:

### 1. **The MCP Positioning** (Primary Message)
**When to use:** Homepage, hero sections, elevator pitches
**Message:**
> "Engrams is an MCP server that brings persistent project memory to AI coding assistants you can't modify — like Cursor, Roo Code, and Claude Desktop."

**Why it works:** Directly addresses the core problem (can't modify tools) and positions Engrams as the solution.

---

### 2. **The Reliability Angle** (Differentiation)
**When to use:** Addressing MCP concerns, technical discussions
**Message:**
> "Unlike MCP servers that depend on tool calling, Engrams is a data provider. It automatically surfaces relevant context without requiring the AI to remember to call tools."

**Why it works:** Removes a major concern about MCP reliability by showing Engrams is fundamentally different.

---

### 3. **The Security Story** (Trust Building)
**When to use:** Enterprise conversations, privacy-conscious audiences
**Message:**
> "Engrams is local-first. Your project knowledge lives in your workspace, never in the cloud. Structured data prevents prompt injection. GDPR/HIPAA compliant by design."

**Why it works:** Addresses three concerns at once: privacy, security, compliance.

---

### 4. **The Cost Efficiency Proof** (ROI)
**When to use:** Cost-conscious teams, scaling discussions
**Message:**
> "Context budgeting reduces token usage by 60-89%. A medium project saves $13.50/month. Large projects save $75/month. Costs stay flat as your project grows."

**Why it works:** Concrete numbers prove the value. Scaling benefit is compelling.

---

### 5. **The Simplicity Promise** (Adoption)
**When to use:** Getting started, onboarding, developer-focused messaging
**Message:**
> "5-minute setup. No coding required. Pre-built governance, semantic search, context budgeting, and knowledge graphs. Everything you'd build in a custom MCP server, ready to use."

**Why it works:** Removes friction. Shows value without effort.

---

### 6. **The Enterprise Assurance** (Maturity)
**When to use:** Enterprise sales, team governance discussions
**Message:**
> "Team-level governance with conflict detection. Amendment workflows for exceptions. Complete audit trails. Built for teams that need to enforce standards while allowing flexibility."

**Why it works:** Addresses maturity concerns. Shows it's designed for real teams.

## Messaging Dos and Don'ts

### DO:
- **Lead with the problem:** "AI tools forget context between conversations"
- **Position as the solution:** "Engrams is an MCP server that fixes this"
- **Use concrete numbers:** "$13.50/month savings for medium projects"
- **Show the comparison:** "5 minutes vs. days/weeks for custom MCP"
- **Emphasize local-first:** "Your data never leaves your machine"
- **Highlight pre-built features:** "Governance, budgeting, search, all included"

### DON'T:
- **Oversell reliability:** Avoid "100% reliable" — say "doesn't depend on tool calling"
- **Claim it replaces everything:** It's a complement to markdown docs, not a replacement
- **Ignore the setup:** Be honest about the 5-minute setup requirement
- **Downplay competitors:** Focus on Engrams' strengths, not others' weaknesses
- **Use technical jargon:** Explain MCP, embeddings, budgeting in plain language
- **Make unsupported claims:** Stick to documented features and real cost comparisons

## Messaging by Channel

### Website & Documentation
**Tone:** Professional, evidence-based, detailed
**Focus:** Comprehensive positioning, real-world examples, cost comparisons
**Length:** 500-2000 words per page
**Examples:** `why-engrams-mcp.astro`, `security-model.astro`, `cost-efficiency.astro`

### Social Media (Twitter/X, LinkedIn)
**Tone:** Conversational, punchy, benefit-focused
**Focus:** One key message per post
**Length:** 280 characters (Twitter), 1-3 sentences (LinkedIn)
**Examples:**
- "Your AI forgets everything between conversations. Engrams fixes that. As an MCP server, it gives Cursor, Roo Code, and Claude persistent memory of your project decisions. 5-minute setup. No coding required."
- "Context budgeting: Engrams selects only relevant information for each task. Result: 60-89% token cost reduction. Larger projects save more. Costs stay flat as you grow."

### Email & Outreach
**Tone:** Personal, problem-focused, benefit-driven
**Focus:** Specific pain point + solution + proof
**Length:** 150-300 words
**Example:**
> "Hi [Name],
>
> I noticed you're using [Cursor/Roo Code/Claude Desktop]. If your team is frustrated with your AI forgetting project decisions between conversations, Engrams might help.
>

>

### Sales Conversations
**Tone:** Consultative, problem-solving, evidence-based
**Focus:** Their specific pain point → Engrams solution → Proof
**Length:** Conversational, 5-10 minute pitch
**Structure:**
1. Acknowledge their pain: "So your AI keeps forgetting your team's decisions?"
2. Explain the problem: "That's because AI tools don't have persistent memory"
3. Introduce the solution: "Engrams is an MCP server that fixes this"
4. Show the proof: "Teams save 60% on token costs and enforce standards"

## Handling Objections

### "Why not just use custom instructions?"
**Response:** "Custom instructions are great for one-off guidance, but they don't scale. Engrams provides structured, searchable project knowledge that grows with your team. Plus, it enforces team standards through governance."

### "Isn't this just a RAG system?"
**Response:** "Engrams is a specialized RAG system optimized for project knowledge. It includes features (governance, bindings, budgeting) that general RAG systems don't have. And it's simpler to set up."

### "What if we need to modify the MCP server?"
**Response:** "Engrams is open source. You can fork it and customize it. But most teams find the pre-built features (governance, budgeting, semantic search) cover their needs without modification."

### "How is this different from a knowledge base?"
**Response:** "Knowledge bases are passive. Engrams is active. It automatically surfaces relevant context to your AI based on what you're working on. Plus, it enforces team standards through governance."

### "What about data privacy?"
**Response:** "Engrams is local-first. Your data lives in your workspace, never in the cloud. No external API calls, no third-party dependencies. GDPR/HIPAA compliant by design."

### "Is this production-ready?"
**Response:** "Yes. Engrams includes team governance, conflict detection, amendment workflows, complete audit trails, and compliance support. It's designed for enterprise teams."

## Messaging Checklist

Before publishing any Engrams messaging, verify:

- [ ] **Problem is clear:** Why does the audience need this?
- [ ] **Solution is obvious:** How does Engrams solve it?
- [ ] **Proof is provided:** Numbers, comparisons, or examples?
- [ ] **Tone is appropriate:** Professional but accessible?
- [ ] **Call-to-action is clear:** What should they do next?
- [ ] **Differentiation is evident:** Why Engrams vs. alternatives?
- [ ] **No overselling:** Claims are supported by documentation?
- [ ] **Consistent with brand:** Aligns with other messaging?
